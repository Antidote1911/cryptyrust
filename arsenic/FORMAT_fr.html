✦ ARSENIC V1

Format de fichier chiffré

Spécification complète du format binaire .arsn, octet par octet. Chiffrement AEAD, arbre Merkle BLAKE3, KEM hybride post-quantique X25519 + ML-KEM-768.

Magic 41 52 53 4E
Version 00 01
Extension .arsn
En-tête min. 232 octets
Keyslot hybride 1 180 octets
Post-quantique ML-KEM-768

1 Vue d'ensemble

┌──────────────────────────────────────────────────────┐  ← offset 0
│  En-tête  (taille variable)                         │  longueur = header_total_size
├──────────────────────────────────────────────────────┤  ← offset header_total_size
│  Payload chiffré  (blocs AEAD consécutifs)           │
└──────────────────────────────────────────────────────┘

Le champ header_total_size (u32 LE à l'offset 0x09) encode la longueur exacte de l'en-tête.

2 Structure de l'en-tête

┌─────────────────────────────────────────────────┐  offset 0x00
│  Section pré-MAC    77 octets                 │
├─────────────────────────────────────────────────┤  offset 0x4D
│  HeaderMAC         32 octets                 │  BLAKE3_keyed_hash(KEK, pré-MAC)
├─────────────────────────────────────────────────┤  offset 0x6D  (PUB_HEADER_LEN = 109)
│  Région d'enveloppe variable                  │
└─────────────────────────────────────────────────┘  offset header_total_size

3 Section pré-MAC (0x00 – 0x4C, 77 octets)

OffsetTailleChampDescription
0x004magic41 52 53 4E — "ARSN"
0x042version00 01
0x061kdf_id01 = Argon2id
0x071hdr_cipher_idChiffrement enveloppe — §10
0x081pld_cipher_idChiffrement payload — §10

0x0D16saltSel Argon2id (16 octets aléatoires)
0x1D4t_costu32 LE — itérations Argon2id

0x254p_costu32 LE — parallélisme Argon2id

0x4112kek_nonceNonce AEAD du keyslot symétrique
Total section pré-MAC : 77 octets (PRE_MAC_LEN = 0x4D).

4 HeaderMAC (0x4D – 0x6C, 32 octets)

KEK       = Argon2id(password, salt, t_cost, m_cost, p_cost)  → 32 oct.
HeaderMAC = BLAKE3_keyed_hash( KEK[32], pré_mac[77] )         → 32 oct.

BLAKE3 est utilisé partout dans le format ; le HeaderMAC l'adopte par cohérence. blake3::Hash::eq est à temps constant.

Fin de l'en-tête public : 109 octets (PUB_HEADER_LEN = 0x6D).

5 Région d'enveloppe (offset 0x6D)

┌─────────────────────────────────────────────────────────────┐
│  WrappedDEK symétrique      48 octets  (offset 0x6D)       │
├─────────────────────────────────────────────────────────────┤
│  hybrid_768_count  (u32 LE) 4 octets  (offset 0x9D)       │
├─────────────────────────────────────────────────────────────┤
│  Keyslot ML-KEM-768 #0  1 180 octets  ┐  × N             │
├─────────────────────────────────────────────────────────────┤
│  hybrid_1024_count (u32 LE) 4 octets                      │
├─────────────────────────────────────────────────────────────┤
│  Keyslot ML-KEM-1024 #0 1 660 octets  ┐  × M             │
├─────────────────────────────────────────────────────────────┤
│  ProtectedMetadata          variable   (TLV_len + 16)      │
├─────────────────────────────────────────────────────────────┤
│  sig_present                 1 octet  0x00 ou 0x01        │
│  [clé vérif. ML-DSA-65    1 952 octets  ┐ si sig=0x01]   │
│  [signature ML-DSA-65      3 309 octets  ┘            ]   │
└─────────────────────────────────────────────────────────────┘

5.1 — WrappedDEK symétrique (48 octets)

WrappedDEK = AEAD_hdr( KEK[32], nonce_env(kek_nonce), "arsenic-v1-wrapped-dek", DEK[32] )
           = ciphertext[32] ‖ tag[16]

5.2 — Compteurs

hybrid_768_count (u32 LE) : nombre N de keyslots ML-KEM-768.
hybrid_1024_count (u32 LE) : nombre M de keyslots ML-KEM-1024. Les deux valent 0 si aucun destinataire asymétrique.

5.3 — Keyslot ML-KEM-768 X25519 + ML-KEM-768 (1 180 octets)

OffsetTailleChampDescription
032ephemeral_x25519Clé publique X25519 éphémère
321088mlkem_ciphertextCiphertext ML-KEM-768
112012kek_nonceNonce AEAD
113248wrapped_dekAEAD(wrapping_key, kek_nonce, aad, DEK)
 
// Chiffrement
eph_x25519_sk        ← CSPRNG OS
ss_x25519            ← X25519_ECDH(eph_x25519_sk, recipient.x25519_pk)
(mlkem_ct, ss_mlkem) ← ML-KEM-768.Encaps(recipient.mlkem_768_ek, CSPRNG OS)
wrapping_key         ← BLAKE3_derive_key("Arsenic Hybrid KEM",
                          eph_pk[32] ‖ mlkem_ct[1088] ‖ ss_x25519[32] ‖ ss_mlkem[32])
wrapped_dek           ← AEAD_hdr(wrapping_key, kek_nonce, "arsenic-v1-hybrid-wrapped-dek", DEK)
// Déchiffrement
ss_x25519            ← X25519_ECDH(recipient_x25519_sk, eph_x25519_pk)
ss_mlkem             ← ML-KEM-768.Decaps(recipient_mlkem_768_seed, mlkem_ct)
DEK                  ← AEAD_hdr_decrypt(wrapping_key, kek_nonce, "arsenic-v1-hybrid-wrapped-dek", wrapped_dek)
Seeds indépendants (depuis v1.5.0) : seeds X25519 et ML-KEM générés indépendamment par le CSPRNG de l'OS et stockés séparément dans le fichier .key. Les anciens fichiers dérivent le seed ML-KEM via BLAKE3 (compat).

5.4 — Keyslot ML-KEM-1024 X25519 + ML-KEM-1024 (1 660 octets)

Niveau NIST 5, ~256 bits quantiques. Même structure que §5.3 avec un ciphertext ML-KEM-1024 de 1568 octets et un contexte BLAKE3 distinct.

OffsetTailleChampDescription
032ephemeral_x25519Clé publique X25519 éphémère
321568mlkem_ciphertextCiphertext ML-KEM-1024
160012kek_nonceNonce AEAD
161248wrapped_dekAEAD(wrapping_key, kek_nonce, aad, DEK)

Contexte BLAKE3 : "Arsenic Hybrid KEM 1024" (au lieu de "Arsenic Hybrid KEM").

5.5 — ProtectedMetadata

MetaKey[32]   ← BLAKE3_derive_key("Arsenic V1 Metadata Key", DEK)
MetaNonce[12] ← BLAKE3_derive_key("Arsenic V1 Meta Nonce",   DEK)[0..12]
ProtectedMetadata = AEAD_hdr(MetaKey, nonce_env(MetaNonce), "arsenic-v1-protected-meta", meta_tlv)

TLV obligatoires (50 octets)

TagLongueurValeur
0x0232MerkleRoot — racine BLAKE3
0x038OriginalSize (u64 LE)
0x051BlockSizeId
0x061MerkleAlgoId = 0x01
Tag 0x04 (CompressedSize) supprimé — toujours égal à OriginalSize.

5.6 — Région de signature

Les derniers octets de l'enveloppe encodent une signature ML-DSA-65 optionnelle (NIST FIPS 204, ~192 bits quantiques).

Valeur de sig_presentSignification
0x00Aucune signature — fin de l'en-tête
0x01Signature ML-DSA-65 : verifying_key[1952] ‖ signature[3309]

Message signé = pre_mac[77] (tous les paramètres KDF, IDs chiffrements, nonces). La vérification est automatique au déchiffrement ; une non-concordance est une erreur fatale.

8 Payload chiffré

Le payload commence à l'offset exact header_total_size. Blocs AEAD indépendants, déchiffrables en parallèle.

Taille de bloc

BlockSizeIdTaille plaintextCondition
0x014 194 304 oct. (4 Mio)Fichiers < 4 Gio
0x0233 554 432 oct. (32 Mio)Fichiers ≥ 4 Gio

Clés et nonces de bloc

block_key_i[32]   ← BLAKE3_keyed_hash(DEK, i.to_le_bytes()[8])
material[32]      ← file_base_nonce[24] ‖ i.to_le_bytes()[8]
block_nonce_i[24] ← BLAKE3_derive_key("Arsenic V1 Block Nonce", material)[0..24]
aad_i[8]           ← i.to_le_bytes()  (u64 LE)

Sur disque

bloc_0[N₀+16] ‖ bloc_1[N₁+16] ‖ … ‖ bloc_k[Nₖ+16]
où  bloc_i = AEAD_pld(block_key_i, block_nonce_i_tronqué, aad_i, plaintext_i)

9 Arbre de Merkle

leaf_i[32] ← BLAKE3_derive_key("Arsenic V1 Merkle Leaf v1",  bloc_chiffré_i)
node(g, d)  ← BLAKE3_derive_key("Arsenic V1 Merkle Node v1", g[32] ‖ d[32])
Vérification avant toute écriture : tous les blocs sont déchiffrés, la racine est recalculée et comparée. Aucun octet de plaintext n'est écrit avant que la vérification soit passée.

10 Chiffrements AEAD

cipher_idAlgorithmeNonce natif
0x02Deoxys-II-25615 oct.
0x03XChaCha20-Poly130524 oct.
0x04AES-256-GCM-SIV12 oct.

Le nonce d'enveloppe de 12 octets est étendu selon le chiffrement ; le nonce de bloc de 24 octets est tronqué. Tag AEAD = 16 octets pour tous les algorithmes.

11 Chaîne de dérivation

mot_de_passe │ └── Argon2id(t/m/p, salt) → KEK[32] ├── BLAKE3_keyed_hash(KEK, pré_mac[77]) → HeaderMAC[32] └── AEAD_hdr(KEK, ..., "arsenic-v1-wrapped-dek", DEK) → WrappedDEK[48] CSPRNG OS → DEK[32] ├── BLAKE3_derive_key("Arsenic V1 Metadata Key", DEK) → MetaKey ├── BLAKE3_derive_key("Arsenic V1 Meta Nonce", DEK)[0..12] → MetaNonce ├── Pour bloc i: │ BLAKE3_keyed_hash(DEK, i.to_le_bytes()) → block_key_i │ BLAKE3_derive_key("Arsenic V1 Block Nonce", file_base_nonce‖i)[0..24] → block_nonce_i ├── Pour keyslot ML-KEM-768 j: │ CSPRNG OS → eph_x25519_sk → eph_x25519_pk │ X25519_ECDH(eph_x25519_sk, recipient.x25519_pk) → ss_x25519 │ ML-KEM-768.Encaps(recipient.mlkem_768_ek, CSPRNG OS) → (mlkem_ct, ss_mlkem) │ BLAKE3_derive_key("Arsenic Hybrid KEM", eph_pk‖ct‖ss_x25519‖ss_mlkem) → wrapping_key_j │ AEAD_hdr(wrapping_key_j, ..., "arsenic-v1-hybrid-wrapped-dek", DEK) → wrapped_dek_j └── Pour keyslot ML-KEM-1024 k: (idem — ML-KEM-1024, mlkem_ct[1568], contexte "Arsenic Hybrid KEM 1024") [optionnel — si clé de signature fournie] ML-DSA-65.Sign(signing_key_seed[32], pré_mac[77]) → signature[3309] + verifying_key[1952] annexée à l'en-tête

13 Changement de mot de passe

Seul le WrappedDEK (48 octets) est re-chiffré. Payload, ProtectedMetadata et keyslots hybrides ne changent pas.

Atomicité garantie par un fichier .bak fsynced (données + entrée répertoire) avant l'écriture en place.
