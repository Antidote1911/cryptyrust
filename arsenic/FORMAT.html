✦ ARSENIC V1

Encrypted File Format

Complete specification of the .arsn binary format, byte by byte. AEAD encryption, BLAKE3 Merkle tree, post-quantum hybrid KEM X25519 + ML-KEM-768.

Magic 41 52 53 4E
Version 00 01
Extension .arsn
Min. header 232 bytes
Hybrid keyslot 1 180 bytes
Post-quantum ML-KEM-768

1 Overview

┌──────────────────────────────────────────────────────┐  ← offset 0
│  Header  (variable size)                            │  length = header_total_size
├──────────────────────────────────────────────────────┤  ← offset header_total_size
│  Encrypted payload  (consecutive AEAD blocks)       │
└──────────────────────────────────────────────────────┘

The header_total_size field (u32 LE at offset 0x09) encodes the exact header length.

2 Header Structure

┌─────────────────────────────────────────────────┐  offset 0x00
│  Pre-MAC section    77 bytes                  │
├─────────────────────────────────────────────────┤  offset 0x4D
│  HeaderMAC          32 bytes                  │  BLAKE3_keyed_hash(KEK, pre-MAC)
├─────────────────────────────────────────────────┤  offset 0x6D  (PUB_HEADER_LEN = 109)
│  Envelope region    variable                  │
└─────────────────────────────────────────────────┘  offset header_total_size

Total pre-MAC section: 77 bytes (PRE_MAC_LEN = 0x4D).

4 HeaderMAC (0x4D – 0x6C, 32 bytes)

KEK       = Argon2id(password, salt, t_cost, m_cost, p_cost)  → 32 bytes
HeaderMAC = BLAKE3_keyed_hash( KEK[32], pre_mac[77] )         → 32 bytes

BLAKE3 is used throughout the format; the HeaderMAC adopts it for consistency. blake3::Hash::eq comparison is constant-time.

End of public header: 109 bytes (PUB_HEADER_LEN = 0x6D).

5 Envelope Region (offset 0x6D)

┌─────────────────────────────────────────────────────────────┐
│  Symmetric WrappedDEK       48 bytes  (offset 0x6D)        │
├─────────────────────────────────────────────────────────────┤
│  hybrid_768_count  (u32 LE)  4 bytes  (offset 0x9D)        │
├─────────────────────────────────────────────────────────────┤
│  ML-KEM-768 keyslot #0   1 180 bytes  ┐  × N              │
├─────────────────────────────────────────────────────────────┤
│  hybrid_1024_count (u32 LE)  4 bytes                       │
├─────────────────────────────────────────────────────────────┤
│  ML-KEM-1024 keyslot #0  1 660 bytes  ┐  × M              │
├─────────────────────────────────────────────────────────────┤
│  ProtectedMetadata           variable   (TLV_len + 16)      │
├─────────────────────────────────────────────────────────────┤
│  sig_present                  1 byte   0x00 or 0x01        │
│  [ML-DSA-65 verifying key  1 952 bytes  ┐ if sig=0x01]    │
│  [ML-DSA-65 signature      3 309 bytes  ┘             ]    │
└─────────────────────────────────────────────────────────────┘

The sender selects one KEM level per file (all keyslots either ML-KEM-768 or ML-KEM-1024).

5.1 — Symmetric WrappedDEK (48 bytes)

WrappedDEK = AEAD_hdr( KEK[32], nonce_env(kek_nonce), "arsenic-v1-wrapped-dek", DEK[32] )
           = ciphertext[32] ‖ tag[16]

5.2 — Counters

hybrid_768_count (u32 LE): number N of ML-KEM-768 keyslots.
hybrid_1024_count (u32 LE): number M of ML-KEM-1024 keyslots. Both 0 if no asymmetric recipients.

5.3 — ML-KEM-768 Hybrid Keyslot X25519 + ML-KEM-768 (1 180 bytes)

OffsetSizeFieldDescription
032ephemeral_x25519Ephemeral X25519 public key

eph_x25519_sk        ← OS random
eph_x25519_pk        ← X25519Public(eph_x25519_sk)
ss_x25519            ← X25519_ECDH(eph_x25519_sk, recipient.x25519_pk)
m[32]                ← OS random
(mlkem_ct, ss_mlkem) ← ML-KEM-768.Encaps(recipient.mlkem_768_ek, m)
wrapping_key         ← BLAKE3_derive_key("Arsenic Hybrid KEM",
                          eph_x25519_pk[32] ‖ mlkem_ct[1088] ‖ ss_x25519[32] ‖ ss_mlkem[32])
wrapped_dek           ← AEAD_hdr(wrapping_key, kek_nonce, "arsenic-v1-hybrid-wrapped-dek", DEK)
// Decryption
ss_x25519            ← X25519_ECDH(recipient_x25519_sk, eph_x25519_pk)
ss_mlkem             ← ML-KEM-768.Decaps(recipient_mlkem_768_seed, mlkem_ct)
DEK                  ← AEAD_hdr_decrypt(wrapping_key, kek_nonce, "arsenic-v1-hybrid-wrapped-dek", wrapped_dek)
Independent seeds (since v1.5.0): X25519 and ML-KEM seeds are generated independently from the OS CSPRNG and stored separately in the .key file. Legacy files derive the ML-KEM seed via BLAKE3 for backward compat.

5.4 — ML-KEM-1024 Hybrid Keyslot X25519 + ML-KEM-1024 (1 660 bytes)

NIST Level 5, ~256-bit quantum security. Same structure as §5.3 with a 1568-byte ML-KEM-1024 ciphertext and a distinct BLAKE3 context.

OffsetSizeFieldDescription

5.5 — ProtectedMetadata

MetaKey[32]   ← BLAKE3_derive_key("Arsenic V1 Metadata Key", DEK)
MetaNonce[12] ← BLAKE3_derive_key("Arsenic V1 Meta Nonce",   DEK)[0..12]
ProtectedMetadata = AEAD_hdr(MetaKey, nonce_env(MetaNonce), "arsenic-v1-protected-meta", meta_tlv)

Mandatory TLV fields (50 bytes)

TagLengthValue
0x0232MerkleRoot — BLAKE3 root
0x038OriginalSize (u64 LE)
0x051BlockSizeId
0x061MerkleAlgoId = 0x01
Tag 0x04 (CompressedSize) was removed — it always equalled OriginalSize.

5.6 — Signature Region

The last bytes of the envelope encode an optional ML-DSA-65 signature (NIST FIPS 204, ~192-bit quantum security).

Value of sig_presentMeaning
0x00No signature — end of header
0x01ML-DSA-65 signature follows: verifying_key[1952] ‖ signature[3309]

Signed message = pre_mac[77] (all KDF params, cipher IDs, nonces). Verification is automatic on decryption; a mismatch is a hard error. Signers use a separate .sigkey file (32-byte ML-DSA seed).

8 Encrypted Payload

The payload starts at exact offset header_total_size. Independent AEAD blocks, decryptable in parallel.

Block size

BlockSizeIdPlaintext sizeCondition
0x014 194 304 bytes (4 MiB)Files < 4 GiB
0x0233 554 432 bytes (32 MiB)Files ≥ 4 GiB

Block keys and nonces

block_key_i[32]   ← BLAKE3_keyed_hash(DEK, i.to_le_bytes()[8])
material[32]      ← file_base_nonce[24] ‖ i.to_le_bytes()[8]
block_nonce_i[24] ← BLAKE3_derive_key("Arsenic V1 Block Nonce", material)[0..24]
aad_i[8]           ← i.to_le_bytes()  (u64 LE)

On-disk layout

block_0[N₀+16] ‖ block_1[N₁+16] ‖ … ‖ block_k[Nₖ+16]
where  block_i = AEAD_pld(block_key_i, block_nonce_i_truncated, aad_i, plaintext_i)

9 Merkle Tree

leaf_i[32] ← BLAKE3_derive_key("Arsenic V1 Merkle Leaf v1",  encrypted_block_i)
node(l, r)  ← BLAKE3_derive_key("Arsenic V1 Merkle Node v1", l[32] ‖ r[32])
Verification before any write: all blocks are decrypted, the root is recomputed and compared. No byte of plaintext is written until verification passes.

10 AEAD Ciphers

cipher_idAlgorithmNative nonce
0x02Deoxys-II-25615 bytes
0x03XChaCha20-Poly130524 bytes
0x04AES-256-GCM-SIV12 bytes

The 12-byte envelope nonce is expanded per cipher; the 24-byte block nonce is truncated. AEAD tag = 16 bytes for all algorithms.

11 Derivation Chain

password │ └── Argon2id(t/m/p, salt) → KEK[32] ├── BLAKE3_keyed_hash(KEK, pre_mac[77]) → HeaderMAC[32] └── AEAD_hdr(KEK, ..., "arsenic-v1-wrapped-dek", DEK) → WrappedDEK[48] OS random → DEK[32] ├── BLAKE3_derive_key("Arsenic V1 Metadata Key", DEK) → MetaKey ├── BLAKE3_derive_key("Arsenic V1 Meta Nonce", DEK)[0..12] → MetaNonce ├── For block i: │ BLAKE3_keyed_hash(DEK, i.to_le_bytes()) → block_key_i │ BLAKE3_derive_key("Arsenic V1 Block Nonce", file_base_nonce‖i)[0..24] → block_nonce_i ├── For ML-KEM-768 keyslot j: │ OS random → eph_x25519_sk → eph_x25519_pk │ X25519_ECDH(eph_x25519_sk, recipient.x25519_pk) → ss_x25519 │ ML-KEM-768.Encaps(recipient.mlkem_768_ek, OS random) → (mlkem_ct, ss_mlkem) │ BLAKE3_derive_key("Arsenic Hybrid KEM", eph_pk‖ct‖ss_x25519‖ss_mlkem) → wrapping_key_j │ AEAD_hdr(wrapping_key_j, ..., "arsenic-v1-hybrid-wrapped-dek", DEK) → wrapped_dek_j └── For ML-KEM-1024 keyslot k: (same — ML-KEM-1024, mlkem_ct[1568], context "Arsenic Hybrid KEM 1024") [optional — if signing_key set] ML-DSA-65.Sign(signing_key_seed[32], pre_mac[77]) → signature[3309] + verifying_key[1952] appended to header

13 Password Change

Only the WrappedDEK (48 bytes) is re-encrypted. Payload, ProtectedMetadata and hybrid keyslots are unchanged.

Atomicity guaranteed by a fsynced .bak file (data + directory entry) before any in-place write.
